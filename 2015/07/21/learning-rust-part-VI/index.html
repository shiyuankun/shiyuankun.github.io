rust学习笔记(六)

可变性(Mutability)

rust的可变性已经提到过很多了，差不多就是这样子

let mut x = 5;
let y = &mut x;

但这次要多一个概念，即内部和外部可变性。

我们在讨论可变性的时候，其实是在讨论外部可变性，比如

use std::sync::Arc;

let x = Arc::new(5);
let y = x.clone();

这样是可以的。因为符合了以下原则

  1. 对一个资源的一个或者多个引用(&T)
  2. 同时只能有一个可变引用(&mut T)

这就是为什么上面那段代码可行，因为任何时候T.clone()给出的都是T的不可变引用(&T)

但下面这段就不行了

use std::cell::RefCell;
let x = RefCell::new(42);
let y = x.borrow_mut();
let z = x.borrow_mut();

这里，y已经借出了一个&mut T，所以z再借出就报错panic了。

由于可变性是借(&mut)或者绑定(let mut)的一个属性，所以对于结构体不存在某些属性可变某些属性不可变。

struct Point{
    x: i32,
    mut y: i32,
}

这是错的，只能这么操作

struct Point {
    x: i32,
    y: i32,
}
let mut a = Point { x: 5, y: 6 };
a.x = 10;
let b = Point { x: 5, y: 6};
b.x = 10;//这里也错了，因为b的绑定是不可变的

当然，可以利用Cell特性模拟可变

use std::cell::Cell;

struct Point {
    x: i32,
    y: Cell<i32>,
}

let point = Point { x: 5, y: Cell::new(6) };

point.y.set(7);

println!("y: {:?}", point.y);

结构体

结构体的定义和c差不多。

struct Point {
    x: i32,
    y: i32,
}

当你要更新一个结构体的时候，多了一种方法..

struct Point3d {
    x: i32,
    y: i32,
    z: i32,
}

let mut point = Point3d { x: 0, y: 0, z: 0 };
point = Point3d { y: 1, .. point };

这样point中除y以外的数值都会从另一个复制过来

Tuple structs
这个定义类似于tuple和struct结合体

struct Color(i32, i32, i32);
struct Point(i32, i32, i32);

let black = Color(0, 0, 0);
let origin = Point(0, 0, 0);

但不推荐这么写，因为没有变量名字。

但有的时候有奇效，比如

struct Inches(i32);

let length = Inches(10);

let Inches(integer_length) = length;
println!("length is {} inches", integer_length);

这样可以从tuple struct解包出一个数值，有点像python的unpack。

Enums

枚举是一个常见的东西，就不细说了
差不多就是把不同种类的变量类型打包成一个类型，这个类型的数据有各种变量类型的可能。
